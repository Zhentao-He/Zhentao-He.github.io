我的第一篇博客

发表于 2022-07-18 更新于 2025-02-28

简介

一直想搞一个属于自己的博客，趁着当下没有什么必要之事，折腾了一天多，才有了我的这第一篇博客。在这里简单记录一下博客的搭建过程。

框架的选择

搭建个人博客的方式有很多，目前主流的方法可以参考一下b站视频《搭博客到底选什么框架？》。
本人不想掏钱（大部分博客都需花钱要租服务器），并且最近时间多、能折腾，于是选择Hexo+Github方式搭建博客。

搭建过程

具体的过程可以参考b站视频《保姆级Hexo+github搭建个人博客》。不过这是一个2021年8月的视频，视频中的文本文件已经挂掉了，因此我下面再补充一些细节。（原视频已被删除，新的教程参考《听说创客们都有这样一个自己的博客》）

Node.js环境配置的一些“坑”

Node.js的安装很简单，“傻瓜式”安装（一直点NEXT）即可。

但是安装好后会发现，只要在命令行中使用npm命令（如使用npm -v查询版本信息）就会有警告：

这是由于npm和node的版本不对应，网上有很多解决办法。但是“警告可以不管”，其实上后面没发现该警告会产生什么大问题。

安装好Node.js后需要干的事情是，修改npm全局包的默认安装路径和缓存路径[^1]。
这是因为之前的我们“傻瓜式”安装Node.js时把该路径设置在了C盘（C:\Users\your_username\AppData\Roaming\npm和C:\Users\your_username\AppData\Local\npm-cache），如果不想C盘中有太多文件，则需要对它们进行修改。

为方便，通常在Node.js的安装根目录（以x:\xx示意）下创建名为node_global和node_cache的文件夹，并在命令行中分别执行

然后我们还要更新用户环境变量Path中全局包的安装路径的值，即把原有的 C:\Users\your_username\AppData\Roaming\npm改为x:\xx\node_global。
需要补充的是，网上许多Node.js安装教程说还要修改系统变量，其实上我们这里没有这个需要。

这实际上是因为我们上一步修改后的全局安装路径和缓存路径对于普通用户没有完全控制的权限，解决办法见《NodeJS安装、NPM依赖路径及NPM INSTALL ERROR 4048解决》。

博客文章的写作

可以参见Hexo的官方帮助文档。

文章文件的格式为.md，为Markdown文件，有很多很好的Markdown编辑器，我选择的是免费开源的VS Code，可以安装一些强力的插件来满足我们不同的需求[^2]。

Markdownd的语法也容易上手，可参考b站视频《8分钟让你快速掌握Markdown》。
这里再补充一点Markdown语法的细节：

  • 标题命令’#’后要有一空格
  • 无序列表’-‘后也要有一空格
  • Markdown 基本语法中并不包含脚注语法，在Hexo中脚注的显示会有问题，需要安装新的模块[^3]。

[^1]: 修改 npm 全局包的默认安装路径——知乎文章
[^2]: Typora 收费了，你还会继续选择使用它吗？ - 三分恶的回答 - 知乎
[^3]: Hexo 里的脚注插件
